# 去年の振り返り

## お仕事

相変わらず DeNA で働いています。チームを異動し、現在はサーバーエンジニアとして働いています。フレームワークは Ruby on Rails です。うちの Rails はよくあるステレオタイプな Rails とは違って、Rails way を完全無視してハイパフォーマンス Ruby on Rails サーバーです。write heavy なゲームアプリのゲームサーバーとして稼働しています。このゲームサーバーを中心とするエコシステムにおいて、広く浅く様々な経験を積めたと感じています。

- daemon (いわゆる worker) の実装
- MySQL のクエリチューニング
- 重いクエリをキャッシュ (Memcached) へオフロード
- Rails アプデ (7.0 -> 7.1)
- Queue のリプレース (to solid_queue)
- 内製 gem の改修
- 不具合修正系

ざっくりとこんな感じのことをやっていたような...実はあんまりビジネスロジック的な部分は書いていないんですよね。どちらかというと足元を整える系のお仕事をすることが多かったです。

印象に残っているのは Rails アプデですかね。7.0 -> 7.1 のアプデをしたのですが、ActiveRecord の各種メソッドの中身にそれなりに変更が入っていたため、ActiveRecord と密結合している内製の ORM ファサードな gem が火を吹いて非常に大変だった記憶があります。あとは serialize format が 7.1 系になったことで Parallel gem 周りで不具合が起こったり、モンキーパッチの影響で Schema Cache の導入に苦戦したりなど。初陣だったのもあるけれど、明らかに後続の 7.2, 8.0 系へのアップデート担当者より大変だった気がするんだよなぁ...ちゃんと評価してもらいたい。

とりあえずこんな感じの基盤系の業務は自分の嗜好にも合致していて楽しいのですが、個人的には運用系のスキルをより育てていきたいという思いがあります。うちのチームは障害が起きたら (基本は) 遅くとも数十分以内に鎮火させられる程度には運用のプロフェッショナル集団です。自分はまだサーバーサイドのこういった運用スキルには課題がそれなりにある認識です。監視や調査のスキルをより育てていく必要性を感じています。

次にクライアントエンジニアリングのお話。クライアントサイドからは少し離れるつもりだったのですが、これもなんだかんだ結構やりました。基本は協業案件なので、先方が対応が難しそうな部分に対して差分を積んだり、クライアントエンジニア視点でのアドバイスをするようなことが多かったです。サーバーサイドチームとクライアントサイドチームの I/F になるような動きは自分にしかできない立ち回りなので、これからもこの強みは活かしていきたいと思います。また、C++ で SDK を書いたりもしました。レガシィシステムのリファクタリングなので、非常に注意を払いならが進めました。結果として 1 年以上前に埋め込まれていたバグを修正したり、全体的にバグの要因となっていたマクロを消すなどそれなりにやることはやりました。ただこれもクライアント領域ということで一緒くたにされてしまい、あまりここの部分を評価してもらえなかったのは心残りです。私は C# + Unity のゲームアプリケーションのプロフェッショナルではあるが、ネイティブレイヤーのプロフェッショナルではないので評価して欲しかったなぁ...

自己評価で言うと、昨年は 5 段階評価で言うと 4 くらいの頑張りだった気がするな。もっとやれたけれど、それなりにはやったという感じ。もう少し評価の目盛りを細かくすると 3.5 くらいかも？

## 副業

基本は C# + Unity を触っていました。ただクライアント、サーバーどちらも品質があまり高くないので、サーバーサイドも多少コードを追ったり、レビューに入ったりはしました。NestJS で書かれているアプリケーションサーバーで、個人的には書き心地は悪くないです。

Unity は主に基盤の実装が多かったように思います。

- Addressables I/F を利用したミドルウェア
- AssetBundle パイプライン
- UnityIAP I/F を利用したミドルウェア
- HTTP Client を利用した API 基盤の改修
- エラーハンドリングにおけるサーバーサイドを含めた設計とクライアントサイドの実装
- クライアントサイドの全体的な設計

正直クライアントの実力が伸びた感覚はないのが少し残念 (まぁ副業の目的はあくまでお金なので良いっちゃあ良い)。とはいえ、それなりに時間を費やしているので、この辺りも踏まえてサーバー側の実装をやれるよう根回ししていきたい。CTO に直接言ってみるのが良さそうかなぁ... またサーバー側もやるということになれば、単価交渉をしてもいいかもしれないですな。
