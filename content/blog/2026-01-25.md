# 動機

年始に作り始めた新規ゲームを作る上で、認証・認可プロトコルを実装すべく調査をしていたところ、Unity 6.3 LTS から `Platform Toolkit` なるものが登場していることを知った。

https://unity.com/ja/blog/unity-6-3-lts-is-now-available

> アカウント管理、セーブデータ、コントローラー所有権、アチーブメントなど、複数のプラットフォームで動作する重要なゲーム機能を 1 つのコードベースだけで統合できる API により、クロスプラットフォーム開発を簡素化します。

Android, iOS 向けに作ろうと思っているため、ひとまずはこのプラットフォームアカウント ID をゲームのアカウントと紐づけることで、ユーザーを一意に識別するというよくある実装を作る。

# 調査と実装

https://docs.unity3d.com/Packages/com.unity.platformtoolkit@1.0/manual/index.html

これを上から下まで読むところから。

少し読むと、各種プラットフォームごとにパッケージを導入する必要があるとのこと。Android, iOS それぞれに個別で子ドキュメントが存在するので、それも見ていく。
