# 動機

現在 Unity の Prefab 差分をいい感じに見れるツールを作ろうとしています。それに伴って YAML の解析器を自作しようとしており、まずは `Unity YAML` の仕様を理解しようと試みました。

# 実験

[こちら](https://unity.com/ja/blog/engine-platform/) の記事が非常に有用なので、この記事の流れに沿って手元で Unity を開きつつ実際に確かめるといった具合に進めます。ただし、手元で確かめる必要がなさそうなほどシンプルな部分はスキップします。

下記リポジトリにて実験を行います。

https://github.com/hashiiiii/unity-yaml-playground

## [YAML の構造](https://unity.com/ja/blog/engine-platform/understanding-unitys-serialization-language-yaml#:~:text=%E3%81%8C%E3%81%A7%E3%81%8D%E3%81%BE%E3%81%99%E3%80%82-,YAML%20%E3%81%AE%E6%A7%8B%E9%80%A0,-YAML%20%E3%81%AF%E3%80%81%E3%80%8CYAML)

まずは [こちら](https://github.com/hashiiiii/unity-yaml-playground/blob/main/Assets/Cylinder.prefab) のファイルの先頭から見ていきます。

```yml
%YAML 1.1
%TAG !u! tag:unity3d.com,2011:
```

`%YAML 1.1` これは YAML 1.1 に準拠しているということを表します。

https://yaml.org/spec/1.1/current.html#id895631

`!u!` は `tag:unity3d.com,2011:` のマクロ。

```yml
--- !u!1 &5661417331689209760
GameObject:
```

各オブジェクトの定義はこの 2 行のヘッダーで始まります。
`1` は GameObject クラスを表しています。

https://docs.unity3d.com/ja/current/Manual/ClassIDReference.html

ここには、どのクラスが何番に予約されているか記載されています。また、1 行目の `!u!1` の `1` と 2 行目の `GameObject` は常に対応しています。つまり、1 行目に `1` と書かれているにも関わらず、2 行目が `MonoBehaviour` になっているということは起こり得ないです。

`&5661417331689209760` これは `File ID` と呼ばれています。File ID が何かについては後述します。

ちなみに、今回の確認には下記のような Prefab を利用しています。

![](/public/images/blog/2026-07-02/%202026-07-02%2016.30.46.png)

この Cylinder ゲームオブジェクトに付与された ID が `5661417331689209760` だったということです。

Cylinder には `Transform` コンポーネントがアタッチされていますが、これは YAML 上で

```yml
--- !u!4 &2764467885147141891
Transform:
```

このように表現されています。UnityEditor 上で `Debug モード` でインスペクターを見てみると

![](/public/images/blog/2026-07-02/%202026-07-02%2016.44.19.png)

Prefab に書かれている File ID と同じ値がインスペクター上でも確認できます。

> File ID が何かについては後述します。

と先ほど言いましたが、インスペクターにもあるように File ID は `Local Identifier In File` です。つまりこの Cylinder Prefab の中では一意ですが、プロジェクト全体で見ると被っているものが存在し得るということです。

これも後述しますが、プロジェクト全体で一意な ID としては `GUID` というものが存在します。

GUID は `.meta` ファイル内部に記述されています。

https://github.com/hashiiiii/unity-yaml-playground/blob/33eb7782ec3373cf92ae7cba1cb97a1bd19e2a37/Assets/Cylinder.prefab.meta#L2

これが Cylinder.prefab の GUID です。

```yml
--- !u!1 &5661417331689209760
GameObject:
  m_ObjectHideFlags: 0
  m_CorrespondingSourceObject: {fileID: 0}
  m_PrefabInstance: {fileID: 0}
  m_PrefabAsset: {fileID: 0}
  serializedVersion: 6
  m_Component:
  - component: {fileID: 2764467885147141891}
  - component: {fileID: 5393027682369059551}
  - component: {fileID: 5404993141587792236}
  - component: {fileID: 7850239779964803711}
  - component: {fileID: 8297381826739130467}
  m_Layer: 0
  m_Name: Cylinder
  m_TagString: Untagged
  m_Icon: {fileID: 0}
  m_NavMeshLayer: 0
  m_StaticEditorFlags: 0
  m_IsActive: 1
--- !u!4 &2764467885147141891
Transform:
  m_ObjectHideFlags: 0
  m_CorrespondingSourceObject: {fileID: 0}
  m_PrefabInstance: {fileID: 0}
  m_PrefabAsset: {fileID: 0}
  m_GameObject: {fileID: 5661417331689209760}
  serializedVersion: 2
  m_LocalRotation: {x: 0, y: 0, z: 0, w: 1}
  m_LocalPosition: {x: 0.64596, y: 0.8118, z: -1.57}
  m_LocalScale: {x: 1, y: 1, z: 1}
  m_ConstrainProportionsScale: 0
  m_Children: []
  m_Father: {fileID: 0}
  m_LocalEulerAnglesHint: {x: 0, y: 0, z: 0}
```

Cylinder.prefab の GameObject, Transform をざっくりと見ていきましょう。

大体は、インスペクター上で設定された値がそのまま入っていると思います。例えば `m_Layer` は UnityEditor 上での `Layer` が `Default` なので `0`、`m_Name` には、ゲームオブジェクトの名前 `Cylinder` が入っています。

先ほど紹介した `FileID` という文字がフィールドに見られると思います。次の章で扱う参照関係にはこの `FileID` が多用されます。
